# 48‑Hour Delivery Checklist

Use this checklist to ensure the SignalScan report is delivered within 48 hours while maintaining quality and accuracy.

## Day 1 (Hours 0–24)

1. **Confirm Scope & Payment**
   - [ ] Verify payment has cleared.
   - [ ] Confirm domains, subdomains and profiles with the client.
2. **Run Passive Collector**
   - [ ] Validate configuration file.
   - [ ] Run `signalscan_collect.py` and store results.
3. **Manual Customer Journey Review**
   - [ ] Browse the site on a mobile device/emulator.
   - [ ] Document friction points and initial impressions.
4. **Surface & Integrity Analysis**
   - [ ] Review DNS, SPF, DKIM/DMARC, TLS and headers.
   - [ ] Identify any broken links or mixed content.
5. **Narrative & Review Analysis**
   - [ ] Scan recent customer reviews for themes.
   - [ ] Note any unanswered questions or reputational gaps.

## Day 2 (Hours 24–48)

6. **Competitor Comparison**
   - [ ] Review three approved competitors using the same passive checks.
   - [ ] Compare speed, clarity and trust signals.
7. **Draft Findings**
   - [ ] Create JSON entries for each finding using the schema.
   - [ ] Assign impacts, urgency and ease of repair.
8. **Build Priority Matrix**
   - [ ] Score each finding and sort into a table.
9. **Prepare Report**
   - [ ] Assemble the data file for the report template.
   - [ ] Render the HTML report and convert to PDF if required.
10. **Quality Assurance**
   - [ ] Use the QA checklist to review content, evidence and wording.
11. **Delivery**
   - [ ] Save report and evidence to the client folder.
   - [ ] Send report via secure channel with short summary.
   - [ ] Offer a walkthrough call to discuss next steps.

Completing each step on schedule helps ensure a consistent, professional and timely client experience.
